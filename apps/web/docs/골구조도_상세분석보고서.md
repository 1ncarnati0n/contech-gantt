## 4. 수식 분석

### 4.1 수식 개요
- **총 수식 개수**: 5개
- **시트간 참조**: 없음 (모두 같은 시트 내 참조)

### 4.2 수식 상세

#### 4.2.1 D3: 공사기간 표시
```
=P33
```
- **의미**: P33 셀의 값(개월 단위 공사 기간)을 표시
- **용도**: 공사기간을 헤더 영역에 표시

#### 4.2.2 L33: 형틀 합계
```
=SUM(L8:L32)
```
- **의미**: L8부터 L32까지의 형틀 물량 합계
- **용도**: 전체 형틀 물량 합계 계산

#### 4.2.3 M33: 철근 합계
```
=SUM(M8:M32)
```
- **의미**: M8부터 M32까지의 철근 물량 합계
- **용도**: 전체 철근 물량 합계 계산

#### 4.2.4 N33: 콘크리트 합계
```
=SUM(N8:N32)
```
- **의미**: N8부터 N32까지의 콘크리트 물량 합계
- **용도**: 전체 콘크리트 물량 합계 계산

#### 4.2.5 P32: 작업일수 합계
```
=SUM(O7:P31)
```
- **의미**: O7부터 P31까지의 범위 합계
- **참고**: P7~P31은 비어있으므로 실제로는 O7~O31의 합계
- **용도**: 모든 층/구역의 작업일수 합계 계산

#### 4.2.6 P33: 개월 환산
```
=ROUNDUP((15+P32+(P32/6))/30,2)
```
- **의미**: 
  - 기본 15일 + 작업일수 합계(P32) + 여유일수(P32/6)를 30으로 나누어 개월 단위로 환산
  - ROUNDUP 함수로 소수점 둘째 자리까지 올림
- **계산 로직**:
  1. 기본 여유일수: 15일
  2. 총 작업일수: P32
  3. 추가 여유일수: P32/6 (작업일수의 1/6)
  4. 총 일수 = 15 + P32 + (P32/6)
  5. 개월 환산 = 총 일수 / 30
  6. 소수점 둘째 자리까지 올림
- **용도**: 총 공사 기간을 개월 단위로 계산

## 6. 파일 용도 및 활용 방안

### 6.1 주요 용도
1. **공사 일정 관리**: 각 층별/구역별 작업 일수 추적
2. **물량 관리**: 각 층별 형틀, 철근, 콘크리트 물량 추적
3. **총 공사 기간 계산**: 자동으로 개월 단위 공사 기간 산출
4. **간트 차트 기반**: 층별 작업 순서 및 기간 시각화

### 6.2 데이터 입력 패턴
- **B열**: 층수 또는 구역명 입력
- **D~J열**: 구역 코드 또는 구역명 입력
- **L열**: 각 층/구역별 형틀 물량 입력 (M²)
- **M열**: 각 층/구역별 철근 물량 입력 (TON)
- **N열**: 각 층/구역별 콘크리트 물량 입력 (M³)
- **O열**: 각 층/구역별 작업 일수 입력
- **P열**: 자동 계산 (수정 불필요)

### 6.3 애플리케이션 구현 시 고려사항

#### 6.3.1 데이터 구조
- 층별/구역별 작업 일정 및 물량을 관리하는 테이블 구조 필요
- 구역 코드와 구역명을 구분하여 저장
- 형틀, 철근, 콘크리트 물량은 숫자로 저장
- 작업 일수는 숫자로 저장

#### 6.3.2 계산 로직
- 형틀 합계: `SUM(L8:L32)`
- 철근 합계: `SUM(M8:M32)`
- 콘크리트 합계: `SUM(N8:N32)`
- 총 작업일수 합계: `SUM(O7:O31)`
- 개월 환산: `ROUNDUP((15 + 총작업일수 + 총작업일수/6) / 30, 2)`
- 공사기간 표시: 개월 환산 결과를 헤더에 표시

#### 6.3.3 UI 구성
- 층별/구역별 작업 일정 및 물량 입력 폼
- 형틀, 철근, 콘크리트 물량 입력 필드
- 작업 일수 입력 필드
- 물량 합계 및 총 공사 기간 자동 계산 및 표시
- 간트 차트 형태의 시각화 (선택사항)

## 7. 주요 특징

### 7.1 장점
1. **자동 계산**: 총 공사 기간 및 물량 합계가 자동으로 계산됨
2. **구조화된 데이터**: 층별/구역별로 체계적으로 정리
3. **물량 통합 관리**: 형틀, 철근, 콘크리트 물량을 한눈에 확인 가능
4. **간단한 구조**: 복잡한 수식 없이 직관적인 구조

### 7.2 개선 가능 사항
1. **P32 수식 범위**: `=SUM(O7:P31)`에서 P7~P31은 비어있으므로 `=SUM(O7:O31)`로 수정 가능
2. **데이터 검증**: 물량이나 작업 일수가 음수이거나 비정상적으로 큰 값 입력 방지
3. **구역 코드 규칙**: 구역 코드 생성 규칙을 명확히 문서화
4. **단위 표시**: 각 열의 헤더에 단위(M², TON, M³)를 명확히 표시
